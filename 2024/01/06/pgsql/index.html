数字类型

CHARACTER TYPES

  • Char
  • Varchar
  • Text

NUMERIC TYPE

Serial:从1开始的序列数，用于column ids

Smallserial,Serial,Bigserial

Smallint,Integer,Bigint

Decimal，Numberic，Real，Double Precision，Float

BOLLEANTYPES

True, False or Null

True,1,t,y,yes,on

False,0,f,n,no,off

null

DATA / TIME

Date

  • 不管是输入的是什么格式时间都会得到:1974-12-21格式

Time

  • ‘1:30:30 PM’:: TIME WITHOUT TIME ZONE -> 13:30:30
  • 01:30 AM EST -> 01:30-5:00(UTC Format)
  • 01:30 AM PST -> 01:30-8:00
  • 01:30 AM UTC -> 01:30+00:00
  • ‘01:30:30 PM EST’::TIME WITHOUT TIME ZONE -> 13:30:30-5:00

DATE/TIME

Timestamp

  • ‘DEC-21-1974 1:30 PM EST’::TIMESTAMP WITH TIME ZONE -> 1974-12-21 13:30-5:00

Interval

  • 表示一段时间
  • ‘1day’::INTERVAL -> 01:00
  • ‘1 D 1 H 1 M 1 S’::INTERVAL -> 01:01:01:01
  • INTERVAL可以用于加减时间

OTHER TYPE

  • Currency
  • Binary
  • JSON
  • Range
  • Geometric
  • Arrays
  • XML
  • UUID

查询customer表

以下是ChatGPT的解释

SELECT：关键字用于指示要执行一个查询操作。
：通配符表示选择所有列。在这种情况下， 表示选择表中的所有列。
FROM：关键字用于指示要从哪个表中检索数据。
public.customer：表示要从名为 customer 的表中检索数据。public 是表所在的模式（schema）名称。如果没有指定模式名称，默认为 public。

ORDER BY：关键字用于指示排序操作。
id：表示要按照哪个列进行排序，这里是 id 列。
ASC：表示升序排序。如果不指定排序顺序，默认为升序。如果要进行降序排序，可以使用 DESC 关键字。

ALTER TABLE

ChatGPT解释

ALTER TABLE 语句用于修改数据库表的结构，包括列的定义和属性。在您提供的示例中，ALTER TABLE customer ALTER COLUMN sex TYPE sex_type USING sex::sex_type 语句用于修改名为 customer 的表中的 sex 列的数据类型为 sex_type，并使用 sex::sex_type 将现有的数据转换为新的数据类型。

  • ALTER TABLE：关键字用于修改表的结构。
  • customer：表示要修改的表的名称。
  • ALTER COLUMN：关键字用于修改表中的列的定义。
  • sex：表示要修改的列的名称。
  • TYPE sex_type：TYPE 子句用于指定要将列的数据类型修改为 sex_type。
  • USING sex::sex_type：USING 子句用于指定如何将现有的数据转换为新的数据类型。在这里，sex::sex_type 表示将 sex 列的值转换为 sex_type 类型。
